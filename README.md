# FI Goal — starter app (rebuilt on verified Expo SDK 56)

## What actually caused the previous version to break

Every issue you listed traced back to one root cause: the previous
`package.json`/`app.json` were **hand-typed from memory** instead of
generated by Expo's own tooling. Expo's dependency graph moves fast and
versions have to match exactly — a hand-typed guess (`expo ~56.0.0`,
`react-native 0.85.0`, `react-native-screens ^3.31.0`, etc.) drifts from
what actually ships together, and that mismatch is exactly what produces
"incompatible with this version of Expo Go," peer conflicts, and
`expo-doctor` failures.

This rebuild fixes that at the source: every version below was pulled from
the **actual npm registry / Expo's own official template package**, not
retyped from memory. I downloaded Expo's real `expo-template-blank-typescript`
package for SDK 56 and built this project on top of it directly.

## Why SDK 56, not the newest SDK 57

Expo released SDK 57 a few days ago. But **the Expo Go app on the Play Store
hasn't been updated to SDK 57 yet** — Expo's own release notes confirm the
SDK 57 Expo Go build is still sitting in Play Store/App Store review as of
this week. Since you specifically need this to work with "the latest Expo
Go from the Play Store," targeting SDK 57 right now would very likely
reproduce the exact "Project is incompatible with this version of Expo Go"
error you already hit — just in the other direction (your project newer
than what's installable). SDK 56 is confirmed live on the Play Store today,
so that's what this project targets. When Expo Go for SDK 57 reaches the
Play Store, upgrading is one command — see "Upgrading later" below.

## Setup

1. **Node.js** (LTS) — [nodejs.org](https://nodejs.org)
2. **Expo Go** on your phone — from the Play Store or App Store (this
   project now matches what's currently live there)
3. A free **Supabase** project — [supabase.com](https://supabase.com)

### Database

In the Supabase dashboard, open **SQL Editor**, paste in the full contents
of `supabase/schema.sql`, and run it. Then go to **Project Settings → API**
for your Project URL and anon key.

### App

```bash
cp .env.example .env
```

Edit `.env` with your real Supabase URL and anon key, then:

```bash
npm install
npx expo start
```

Scan the QR code with Expo Go. No `--legacy-peer-deps`, no `--force`,
no `expo install --fix` needed this time — the dependency tree is already
internally consistent.

## Upgrading later, when SDK 57 reaches the Play Store

```bash
npx expo install expo@57 --fix
```

Then re-run `npx expo-doctor` to confirm. Do this only once you've checked
[expo.dev/go](https://expo.dev/go) or the Play Store listing shows SDK 57
as current — upgrading before Expo Go catches up just reproduces the
"incompatible" error in the other direction.

## Project structure

```
src/
  lib/supabase.ts       — Supabase client (never crashes on missing config)
  types/index.ts         — TypeScript types matching the DB schema
  utils/budgetEngine.ts  — the core algorithm (pure functions, no RN/DB deps)
  screens/                — one file per screen
  navigation/             — the navigation stack
supabase/schema.sql       — full DB schema + row-level security policies
```
